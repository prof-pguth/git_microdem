DEMIX Pipeline Processing Steps in MICRODEM

DEMIX button on the main program menu, Tiles creation, test and reference DEMs

MICRODEM Setup

  • Download and install MICRODEM
  • Perform  default GDAL installation from OSGeo4W.  This is needed to read some Geotiff DEMs, reproject to WGS84, and has transformation grids to convert vertical datums to EGM2008

Setting up MICRODEM  DEMIX Definitions Databases

  • Samples of these files are included in the installation program, in "c:\microdem\demix_settings\"
  • National datums database
Get HRDEM for Reference DEMs 
  • Download national mapping agency files for reference DEMs
    • For HRDEM that must merge 1x1 km tiles, there are options.  For some countries there is MICRODEM source code that can be modified to create a list of 100 1x1 km tiles to fill a UTM DEMIX tile if the country supports automatic download by sending the name of the file
    • Place files into directories with specified structure
  • EPSG codes for the horizontal and vertical datums are currently hard wired in the MICRODEM source code.  This will eventually moved into a database
Landcover data for test areas. 
  • Currently
    • ESA Land cover 10 m 10 m 11 classes
    • https://viewer.esa-worldcover.org/worldcover/? allows selection of 3x3 degree tiles
    • Place in x:\landcover\esa_world_cover_10m (any drive)
    • Changing this requires require source code changes (1) adding file name recognition for another data set, and (2) adjusting land cover category codes for aggregation in forest and barren super-categories.
  • Originally
    • Copernicus Global Land Cover (GLC100). 
    • Place in x:\landcover\Copernicus_LC100
    • This had a single, global file
Collect and index test/candidate Global DEMs
  • Create directories in the root directory of a drive x:\mapdata\indexed_data\DEMs, with subdirectories for each of the 1" global DEMs
  • Insure that the x:\microdem\demix\demix_dems.dbf is updated
  • Download  Global DEMs for all test areas to be used.  Each is a separate downoad from a different location.  Place files in correct subdirectories; they can be in additional nested subdirectories if desired. 
  • Index map library

Process data, using DEMIX button on the main program menu, Tiles creation, test and reference DEMs.  This performs the following steps:

  1. Completes creating the directory structure for the comparison data
  2. Merge the files in the directory (MICRODEM will call GDAL) if the country requires merges.  This merge file is not saved.
  3. Convert the file (MICRODEM will call GDAL) to WGS84 UTM, EGM2008 or just WGS84 UTM if GDAL does no support the vertical datum shift
  4. Create the test DEMs files for each tile
    • Test DEMs must be in the map library and indexed
    • 10 m landcover must be in a directory "X:\landcover\esa_world_cover_10m"
    • ALOS will be transformed to EGM2008
  5. Mean aggregate to 1 arc second reference DEMs
    • Test DEMs will be merged and clipped to the extent of the high resolution DEMs for each tile
    • Uses Fathom DEM for the SRTM geometry
    • Use ALOS for the ALOS geometry, and the high latitude
    • Use CopDEM for the high latitude areas where CopDEM, but not the derived DTMs, changes the longitudinal spacing

Evaluate criteria, using DEMIX button on the main program menu, Create CSV results for test areas.  This creates a CSV file for each area:

  • Tile characteristics
  • Difference distributions for elevation, slope, and roughness.  This follows the first DEMIX paper (Bielski and others, 2024)
  • FUV for a mixed suite of criteria.  This follows the later DEMIX papers (Guth and others 2024, 2025)
  • All three at once (if there are a lot of areas, each of these can be slow)
  • Partial derivatives and curvature criteria used in DB version 4 still available
Create DBF files, using DEMIX button on the main program menu, Merge CSV results for test areas.  This merges the CSV files for each area.
Manipulate database, in MICRODEM or possibly in the DEMIX Jupyter notebook (not tested with changes in the database, and the notebook may not deal  with the new DEMs)
