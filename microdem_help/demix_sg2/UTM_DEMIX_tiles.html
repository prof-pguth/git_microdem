UTM DEMIX Tiles

By UTM, we mean the projected system used by the national mapping agency.  In addition to UTM, this could be:

  • UKOS grid
  • French Lambert Conic

Delivery files have two geometries:

  • On the UTM (or national equivalent) grid, at even 1000 or 10000 meters, preferably named for the SW corner (the general case, except for USGS which uses NW corner)
  • On the boundaries of the quad or quarter-quad of the national topo map sheets
  • As monster (> 1 GB) tiles. We do not currently support DEMs supplied as huge (> 1 GB) Geotiffs.  Conceptually we would:
    • Use GDAL info to get the limits of the DEM in projected coordinates
    • Find the full 10x10 km areas, with the lower left corner at even 10000 in both x and y
    • Use GDAL to extract the tile in projected coordinates

Final tile names include a two letter country code followed by "_".  Example include "ca_". "fr_".  These must match what is in the MICRODEM source code, and are used for the datum transformations.  This will eventually go into an external database. Where the country supplies data by the first level political subdivision, there is a second two letter code (e.g. Germany)

Handling depends on the details of the mapping agency files

  • Single file is about 10x10 km (US, Canada,Spain): tile name assigned from the files used by the mapping agency
  • Four files for about 10x10 km (Estonia, Finland, Mexico, Netherlands, UK): user responsible to name the DEMIX tile, and put the four files in it.
  • One hundred files for about 10x10 km (France, Denmark, Germany,Switzerland): MICRODEM has code to help generate lists to download the 100 files for some countries, and putting files into the corrrect directories for the tiles.

Probably none of these options are completely automated, and should be regarded as a work in progress.
