Lidar Change Lab

This exercise has several purposes:

  1. Insuring that you can create and interpret figures from lidar point clouds and grids, picking the best display options to show what you want to describe.  You will first have to explore the data to see what it show, and then communicate your results to others.
  2. Practice in writing (hopefully clearly and concisely), and in referring to figures.
  3. Think about what is real, and what might be an artefact of the data.

When looking at imagery, consider:

  1. "Color".  This could be in a non-visible part of the spectrum displayed in the visible, or show something like elevation or change or category.
  2. Texture: vegetation will be rough, unless it's a golf course or parade field; roofs will be smooth
  3. Shape: straight lines are almost always man made.
  4. Size: shadows can help.  Even if you don't have a scale (not an issue in a GIS program), you still have relative clues like the dimensions of a track, soccer field, or softball field.

Deliverables in a single Word Document in Blackboard: 

1.   Create a table showing the changes between 2020 and each of the previous surveys for the area of the Naval Academy. (Alternatively you can use 2004-2011, 2011-2017, 2017-2019, and 2019-2020 as the periods to consider.)  Show major additions and subtractions in separate columns. and a final column with any notes you care to make about what the data quality of the data or the overall changes.  You do not have to repeat the information in the table in your text.

2.  The 2019 dataset has 4-5 times more data than the 2017 and 2020 surveys.  Compare it to those surveys, using at least three figures, one from each of the groups below.  Which of the three data sets would you recommend to other users?  This should probably be 1-2 good paragraphs.

  • Point cloud
  • DEM grid
  • Slice

You do not have to write a full lab report.  In your Word document, start with text.  When you want to refer to a figure, you say "Figure 1 shows...." or "The Naval Academy is on the banks of the Severn River (Figure 1)".  Then put the figure after the next paragraph break; you do not have to worry about blank space if it shows up on the next page.  Below the figure, put a caption: "Figure 1.  Location of the Naval Academy."  Your readers are smart enough that you do not have to say "Figure 1.  This map shows........".  For the table the caption goes above.

If you want to play the academic game correctly, copy this list to your report Word document.  However, your text should stand on its own and not require the directions to make sense, and the directions should not be in what you turn in.

Initially start with File, Remote sensing labs, Annapolis lidar

  • Add LIDAR Point clouds in c:\mapdata\annapolis_lidar. 
    • You want all the files in the directory, and it is easiest to do selection by directory.
    • You can pick multiple clouds at once
  • Zoom the map to the common area of coverage
    • Corner lat long
         NW 38.99597 -76.5062
         NE 38.99597 -76.4748
         SW 38.97304 -76.5062
         SE 38.97304 -76.4748
    • Paste the table above into Excel
    • Save it as a CSV file
    • Close Excel (another program cannot access its open files)
    • Open database to open the CSV file (adjust the file type filter if necessary).  This coverts to a dBase DBF file, which is what GIS programs use.  If you were to make changes, they would be in the DBF and not the CSV file.
    • Zoom to just this region using one of the methods
      • On the DB form, Plot, Zoom to DB coverage
      • Subset & zoom the map to show just this region using the mouse
  • Create 1 m DSM for 3 most recent lidar collections.  For the 2004 data set, create a 2.5 m DSM, and for the 2011 data set, a 2 m DSM.  Save each with a meaningful name in a location where you can find it.
    • Set the grid parameters
      • UTM projection
      • Desired spacing
      • Auto fill holes, gap of 2 pixels
    • Pick DSM creation
  • Create 4 difference maps, 2020 to 2019, 2017, 2011, and 2004, or 2004-2011, 2011-2017, 2017-2019, and 2019-2020.  This is multiple grid map algebra.  Save these with a meaningful name in a location where you can find it.
    • Pick the 2020 DEM first.  This uses its higher resolution grid spacing, and makes change to higher elevation positive.  If you would prefer the opposite convention, you can use single grid map algebra (multiply by one) on Edit, Single grid arithmetic, Math operations, Multiply z values

Open and manipulate maps

  • Open Grid/DEM and pick one of the lidar DSMs or the change maps.
  • LIDAR Point clouds on the map toolbar in c:\mapdata\annapolis_lidar. 
  • Open database (shapefile) if you want the TIGER data, in C:\mapdata\annapolis_data\tl_2022_24003_edges

Explore the data

  • Point cloud
    • Map view, multiple parameters (elevation, intensity, classification, point density)
    • 3D view manipulation (subet the area first)
  • DEM grid--control the grid spacing and the derived surface (DSM, NVS, DTM)
    • Map view, Display parameters (Elevation, Slope, Reflectance/Hillshade..... more)
    • Difference map between two grids map to see changes.  This is called map algebra.
    • OpenGL 3D view
    • Topographic Profiles  or Multiple DEM profiles
  • Slice: EW and NS slices through the point cloud.   (subet the area first) Insure that you go through College Creek to see how the different surveys treat water.
