Create Registration File

  1. REG File button, which will also automatically save the registration points.
  • TIN Mapping: any map will work as long as you enter enough control points.  This uses all the registration points entered.
  • Simple registrations
    • "UTM world": requires that UTM grid lines are lined up north-south and creates a world file.  If this is a valid choice for your data, "Geo" is not valid. For many data sets, neither choice is appropriate.
    • "Geographic world": requires a lat/long graticule lined up north-south and creates a world file.  If this is a valid choice for your data, "UTM" is not valid. For many data sets, neither choice is appropriate.
    • Find optimal rotation registration: this will consider all combinations of three control points, and show you the average registration error and the maximum with the points considered as verification of the results.  The three points with the smallest average error will probably produce the best rotation registration results.
  • Shift prime meridian: move all the longitudes, for instance in you are doing "quad map" and the map uses an old Prime Meridian.  You can enter an arbitrary shift, or use the Ferro value.
  • Lat/long Graphs
  • Export map for Google Earth.  This will create a KML or KMZ file and open it in Google Earth.  This file will be stored in "c:\mapdata\kml" in a new directory, but you can save it from Google Earth with a name you want and a location where you can easily find it.  You can overlay this file on a map (like a DEM) with the Overlay, Imagery (KML, KMZ, world files)
  • Reproject map: This will create a Geotiff. 
  • Insure the Datum and UTM zone are correct. If not, select the Datum button and correct them.
  • The first three options (world files and rotation) use only the first three registration points.  Those points would ideally be in three corners of the map; they should be as far apart as possible, and as far from being on a straight line as possible.  Any points beyond three will be used to estimate the accuracy of the registration.
  • For the world file  register options, you will get a window with statistics. These are in the form y = ax + b, and will show the results for both a UTM and lat/long spacing.
