These tools are designed to be called from the command line, or from scripts.  To call them, MICRODEM

  • creates a command line with the tool name, input and output files, and required parameters.   You may have options to specify the parameters.
  • if required (such as setting up GDAL, or sequentially operating on multiple files), the command is put into a batch file.
  • Where it makes sense, MICRODEM will create multiple batch files which will each process some of the input in parallel if you have multiple processors.  If the operation is disk limited, as opposed to CPU limited, this may not buy any speed improvement.
  • Runs the command or batch file
    • For operations expected to very fast, the "black window" is not shown
    • For longer operations, you can track the program execution in the black window(s).  You will not see any final output from the tool because the command window will close.  The amount of progress shown depends on the program, and can range from nothing to very detailed.
    • When the purpose of running the tool is to get information, the tool's output is redirected into a file which is then opened in a text editor.
  • The command is written to MICRODEM's debug log.  You can get the command line there, paste it into a command prompt window, and see any final output or error messages.  If it is a batch file, the file will be in c:\mapdata\temp, but you must check it before you start MICRODEM again.
  • In most cases, MICRODEM waits for the program to finish, since it needs the output top proceed.  For a few cases, where the output is not needed, you can immediately resume operations in MICRODEM.
