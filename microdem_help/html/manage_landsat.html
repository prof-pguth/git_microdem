Managing Landsat data

  • Insure you get Landsat Collection 2 Level-1 (or are very sure you know what you are doing if you pick something else). 
  • Three options to download via EarthExplorer download directions
    • Get Bundle with all bands, offers the most flexibility and the ability to manipulate the image
      • Scenes will be downloaded as Unix .tar files
      • Put all scenes that you want to use together in a single directory
      • Have no spaces anywhere in the path to the file location
      • Do not rename the files or folders; the program needs them to manipulate the data
      • Use the special option on Compress/Decompress, Landsat Geotiffs from unix tar.  This will keep each scene in its own directory.  You can open multiple scenes together from the single directory.
        • Landsat 9 will have 25 files for each scene (11 bands and various metadata files)
        • Landsat 1 will have 16 files for each scene (4 bands and various metadata files)
        • Other Landsats will have an intermediate number of files
      • Open image.  Using the folder open option allows opening multiple scenes at once.
        • The first time the scene opens, GDAL will convert the Geotiff files for use.  The original files will be recycled.
        • Create histograms for each band, useful for Contrast enhancement
    • Browse images, which are much smaller but cannot be manipulated.  They could be a good choice if:
      • You just want to use the images as a base map, perhaps with a time series, and the default band selection and contrast works.
      • You want to digitize onsceen, for something like a glacier terminus, and the default display works well.
      • If you want to do something where a band ratio like NDVI would differentiate the categories, the browse image would not be a good choice
    • Individual bands. 
      • The program will not know particulars about the band, and cannot do multi-band merges.  You should be very clear about the limitations you are accepting if you get these.
      • Open either:
        • As a DEM/grid.
        • With the File, Open, Open single Landsat band

Handling Problems

If there are problems with the Landsat data, the original TAR files should be in the recycle bin (they were not really deleted, at least if you used the Landsat extraction option).

 

Open the recycle bin.  There is an option to do this under File, Tools, File operations

Search for "*.tar" files

Highlight those you want to restore.

Note the original location, which is where they will be placed.

Repeat the decompression, paying particular attention to use the special option for Landsat.

 

Problem Solution Notes
General issues Update the program  
Got Level 2 data.  It does not have all the bands, and reports missing bands Redownload, and get level 1 This might be working, but if it is, you're not reading this
Cannot see the data Track data locations.

This normally happens when

  • you download data with different path/row locations
  • Combine the early and late Landsats, and the scene coverages are different
TIF files disappeared See if you can recover the TAR files (abve), and decompress again (using the landsat option)  
Placed multiple scenes in the same directory See if you can recover the TAR files (abve), and decompress again (using the landsat option)
