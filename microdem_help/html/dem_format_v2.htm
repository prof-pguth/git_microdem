MICRODEM version 2 DEM Format

tHeaderRecord = packed record   //for a DEM

  • DEMUsed : tDEMAvailable;
  • DEMPrecision : tDEMprecision;
  • DataSpacing : tSpacingUnit;
  • NumCol, {number of columns in data set}
  • NumRow : SmallInt; {number of rows in data set}
  • MaxElev, {max elevation in data set}
  • MinElev, {min elevation in data set}
  • fLatInterval,
  • fLongInterval,
  • oldfSWCornerx,
  • oldfSWCornerY : shortfloat;
  • ElevUnits : tZValueUnit;
  • DefProjName : string[7];
  • SquareID : string2;
  • UTMZone : Byte; {6ø UTM Zone number, USGS/MGRS standard: 1 = W177, 60 = E177}
  • DMAMapDefinition : tDMAMapBasicDefinition;
  • DigitizeDatum : tDigitizeDatum; {sets the datum for the DEM, and it is transformed to the desired local datum for use}
  • LatHemisph : char; {N or S}
  • CreationInformation : string[29];
  • FIPS_Zone : SmallInt;
  • hdfSWCornerx,
  • hdfSWCornerY : Float;
  • FutureUse : array[1..7] of byte;
    end;

 

On Jan 3, 2007, the structure of the headers changed.  Either the oldfSWCornerX and  oldfSWCornerY, or the hdfSWCornerx, and hdfSWCornerY fields will be used.  The other set will be set to 0.  Software reading both types of headers will have to check both fields, and use the non-zero values.  This change was required by finding that the shortfloat, with 7 significant figures, did not "accurately" locate UTM coordinates.
