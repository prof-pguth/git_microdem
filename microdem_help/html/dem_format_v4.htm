MICRODEM version 4 & 5 DEM Format

The MICRODEM format goes back to the mid 1980's, and has evolved as DEMs moved from 16 bit integers.  While the program can read other formats, they are converted to this format for the program's internal representation of a grid.

For interchange with other programs Geotiff is a better (probably mandatory) option. 

  • The only thing not currently in a Geotiff is an indication of what the z values represent, which we add as a note which MICRODEM can interpret and which other programs will ignore.
  • MICRODEM also reads its native format much faster than it does a Geotiff.

MICRODEM format DEMs can now save the Geotiff keys for some map projections (previously only geographic and UTM were allowed).  This is an extension to the v4 format.  This allows creation of derivative grids that are not UTM and not geographic.  Mars DEMs in equirectangular were the initial impetus.

*END		 All number keys are the values from the Geotiff header.  These are intially read from a Geotiff file, and written to the MICRODEM file when derivative grids are created.  The 3075 code defined the projection, its presence alerts MICRODEM to  

The header consists of 1024 ASCII bytes at the beginning of the file.  Each field consists of a key, an "=", the value, and a #13 line termination.  The order of keys is not specified, and more can be added at any time as long as the total size does not exceed the 1024 characters.

Version 4 added the WKT projection in early 2021.  Version 5 was added in  March 2021, to doubled the header size to 2028 to deal with very verbose WKT.

The format was introduced 7/30/2014, to accommodate reading on Android and IOS devices.  Between the problems with byte order and string definitions, it was time for a clean break and an easily extensible format.
