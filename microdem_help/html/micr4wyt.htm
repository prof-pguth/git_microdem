Picking Grid Size

Selecting a grid size (data spacing) for a DEM involves a number of tradeoffs. The grid size directly determines the number of points, which determines the disk space and memory required to manipulate the data set.

As a rough rule of thumb, there should be a digitized data point for every point that will be in the resulting grid. The program using the DEM will interpolate elevations among the grid points, so there is no point in taking a sparse set of digitized points and creating a very fine grid. The MICRODEM program will not use more than 10 digitized points for each grid point to be derived, so you should not waste effort digitizing too many points.

MICRODEM will support a grid size of (at least) 7201x7201. The limit for interpolation is probably 64,000 digitized points. If these limits present problems, digitize in smaller blocks and then attempt to merge the resulting DEM together.
