Managing MICRODEM

  • Update MICRODEM program.  which backs up the old EXE, so you can restore if the newer version breaks something you need.

Hints to help manage multiple copies of  MICRODEM:

  • The program needs essentially every file installed into the directory with the executable.  Some files may be needed only for certain esoteric options, but failure to find files will eventually cause problems if you try putting new EXE files in different directories without copying all the support files.  
  • The key version number is the build number like 2016.5.9.x which will probably change with every build. The build number is the date of compilation, and a sequential number which is probably not very useful.
  • The EXE can be renamed, and multiple EXEs can coexist in the same directory (but each will have its own INI file and default settings). 
  • We do not keep backup copies of old EXE files. We will  try to fix any bugs, but it will be only in the current version of the program, with whatever other new warts it might have.
