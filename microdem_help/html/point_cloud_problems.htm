Point Cloud Problems

  • Hints with large point clouds or wimpy computers to improve performance
  • Insure you do not have an unnecessarily restrictive Filter, which can caused you to see few or no points.

Will not plot

  • An unfortunate number of LAS files do not have the projection set in the LAS file.
  • Follow the procedures in the table below.  Some of these require current LAStools
Try opening LIDAR Point cloud Works, done    
  Fails, look at LAS file's metadata
  • Use MICRODEM's File Metadata option.
  • Use LAStools
  • Look at download site, which often has the metadata.
 If no projection information, assign it with  LAStools Must have either the UTM zone, or the EPSG code, and this must come from the metadata with the file.  Metadata is often an XML file, best viewed in NotePad++.  There is only one correct choice here, and it is whatever the creator of the data set used.  You cannot use this step for Reprojection.  If the data is not now geographic or UTM, you must reproject it for MICRODEM.
    If there is projection information, but it is not geographic of UTM, reproject it Must have the correct UTM zone for the operations area.
       

Elevation colors a single shade

  • Set the range on the Draw tab of the LIDAR Point clouds form.
