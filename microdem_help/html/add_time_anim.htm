Add Time Animation Fields

From Edit button, Date/Time submenu on the table display window. The prepares time fields in the correct KML format for a database.

MICRODEM can create some of these fields on the Date/Time submenu

Set options.  The appropriate value depends on the duration of your data, and what you want to show with the animation:
  • Month and years: will create a monthly animation.  Requires fields "YEAR" and "MONTH".
  • Months in a single year.  This will put all the data in the year 2000, and produce an animation showing the occurrences by month.  This makes sense for many earth science events which relate to the annual climate cycle, and for which you have insufficient data to show the trends with a month and year animation. Requires field "MONTH".
  • Years. Requires field "YEAR".
  • Days:  Requires fields "YEAR", "MONTH" and "DAY".
  • Full time (like 1999-12-31T23:59:59Z (UTC)): Requires fields "YEAR", "MONTH", "DAY" and "TIME_STR".  These will be the "START_DATE", and the "END_DATE" will be the next entry in the table.  The records must be in chronological order.

You can select either UTC, or a local time and specify the offset from GMT (- in the Western Hemisphere).

This will add new fields START_DATE and END_DATE to the database if they do not exist.  If they already exist, the current values will be overwritten.

This requires the fields YEAR (must have values with 4 digits; edit the database if it only has two digits for the years) and MONTH (must use numbers 1 to 12) and perhaps DAYS (1 to 31).

This will enable you to have a KML animation when you export your database as a KML file for Google earth. .
