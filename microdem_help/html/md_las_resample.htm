MICRODEM LAS resampling

Data Manipulation, Resample , LAS files (MD): this will convert the coordinates to geographic, do a datum shift to WGS84 (if required), and then convert the coordinates to UTM, in one step. 

  • Defined to UTM: MICRODEM must be able to recognize the projection, which must be in the LAS file.  This currently works for SPCS files.
  • Change UTM zone
  • Irish grid to UTM--works even if no projection is assigned in the LAS file, and does not look for the projection.  This has not been used since there were major revisions to handling projections.
  • UK OS to UTM--works even if no projection is assigned in the LAS file, and does not look for the projection.  This has not been used since there were major revisions to handling projections.
  • z shift, feet to meters
  • LAS to.... converts a single LAS file to ASCII.  These will use the number of decimals for the coordinates that are set on the Options form.  This is grossly inefficient and should only be used for small areas.
    • XYZ
    • XYZ, RGB
    • XYZ, Class, Intensity
    • GeoJSON-P

These will also set the coordinate precision to cm in the exported LAS files.
