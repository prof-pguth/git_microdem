LAS file prep, create grids

  • Download point cloud files
  • If you got ZIP file, uncompress
  • Check what the data projection is, before you do anything with LAZ files
    • You can use the LAStools lasinfo, on File, Analyze menu, opn either a LAS or LAZ file.  Hopefully the projection information is in the file.
    • If the file does not include projection information (a failure of the producer/distibutor) you can check the metadata on the download site.  Some include the metadata files in their download, others do not.
  • If the files are not UTM or have no assigned projection, you should use LAStools to resample to UTM  and decompress to LAS format in one step.
  • If files are compressed (LAZ in the UTM projection or eLAS in any projection), Decompress  LAS/LAZ--point clouds 
  • If the LAS files will not work in MICRODEM, perhaps because of flaky or no geo-coding in the LAS file, try resampling to UTM (you should have gotten UTM in the first place).
  • Open any map, such as
    • Quick vector map
  • LIDAR Point clouds and pick the LAS files or the directory.
  • If desired, Subset & zoom the map.  Depending on the size of the area and DEM grid size you want to create, you could run out of memory.   The new grids will cover the same area as the base map.  Once you have a good base map, use it to reload the point clouds and keep all grids the exact same size, which will make any map algebra faster.
  • Create grids

Anomalous points in the lidar data will cause the color scale in your grid could be off.  Low noise will mean very few points on the low end of the color scheme, and high noise will mean very few points on the high end.  The net effect will be to squeeze all the elevation colors into a narrow central range.  You can get around this by using the percentile option for the color scale (z Range, Percentiles).
