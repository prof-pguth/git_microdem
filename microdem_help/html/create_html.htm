Create HTML for KML Display

Create an HTML file with text, graphics, and other standard features such as a reference list.  If you do not know how to use any other HTML editor, you can use MS Word or OpenOffice Write (if you use Google Docs, see below, and start early).  These are certainly not optimal HTML editors, and create bloated HTML code.  But if you just want to create HTML without the bother of learning something else, they will work.

  • Make the first two lines in the document something that will make sense when it is put into th table of contents in Google Earth (see examples below).
  • Keep the formatting very simple, since the output will go through several encodings before the final users see it.  You also have limited space as displayed by Google Earth.
  • Figures
    • Put the figures in the text (use the "In line with text" format option), the captions as a line of text below them, and do not try to wrap text around the figures or tables, or use multiple columns. 
    • Make sure there is a blank line after every image.  A good way to insure this is to have a figure caption or explantory text between all figures.
    • Do not try to put two figures side by side, unless you use Paint to merge them into a single image (so then you do not have Fig. 1 and Fig.2, but Fig.1a and Fig.1b).
    • Do not use text boxes for the captions, or use the Word option to insert captions.
    • Word used to degrade the quality of the graphics.  If they are too small for your taste, or the quality is insufficient, you will have to use an HTML editor.  This is now much less of problem.
  • No footnotes
  • Avoid the following characters: &, >, <, ", and probably any other special symbols.
  • Save the file as a word processor document.  If you want to edit it later, use the word processor document and not the HTML.
  • Sometimes Word is not helpful.  You should consider saving multiple versions of the file, stamped either by time ("project_Wed_0900") or version number ("project_v1").
  • Save the file in a directory where you will create a point database, using the save as Web page/HTML option.  Use a file name that has no unusual characters (in particular no spaces), only letters, numbers, and the underscore character.  Make sure the directory name (complete path) has no spaces.  Try something like c:\temp, perhaps in a new subdirectory with nothing else.  Use CamelCase or underscores if you want.
  • Remember that only image names are included in the HTML file, and not the images.  If you want to move the HTML file, you must also move the images in a correct manner--the best solution is not to move anything after you save as HTML, or move the entire directory.  The program must be able to find the images when it packages up the KML file.  The images will be in a subdirectory in the folder with the HTML file.
  • Check the HTML file by viewing it it a web browser.  If there are problems you should fix them in Word, which will allow you to edit the HTML file.
  • MICRODEM will strip out some of the worst of the Word HTML code bloat when it puts the files into the KML file.   You can turn this off (options form, Web tab, KML options) but the HTML display in Google Earth might not like the Word specific HTML.
  • You must close the file in Word before moving to convert it to KML in MICRODEM.

If you want to include an animated GIF:

  • Word will output a single image from the GIF, in GIF format.  You can recognize it if you view the thumbnails for the images.
  • In the images folder for the export, replace the GIF image for the single frame from the animated GIF movie (flooding, OpenGL rotation, annual climate cycle, others) with the complete file.  You must use the same name given by the Word export, so you will have to rename the file.
  • You can also edit the HTML in an HTML editor, but if you can do that you would probably not be using Word to create the HTML.

If you want to use Google Docs, start very early and be ready for problems

  • Create the document
  • Make sure that EVERY carriage return is done twice, and that you have a real blank line on either side of every image.
  • File, Download, Web page (html, zipped)
  • Move the zip file (or download it there in the first place) to a directory with nothing else
  • Unzip
  • The file will have an HTML extension (not just HTM), and the images will be in an "images" folder.
  • Some students experienced severe problems with this HTML file, before covid.  This may be solved, as Nov 2021 testing showed no problems with a carefully created document.  Web editors like MS Expression Web 4 open the file as a mass of overlapping images and text. The HTML itself is miserably formatted, making it very hard to parse to see what the problem is.  MICRODEM cannot turn it into a KML file. To resolve, you must open in Word, and resave the HTML, which removes whatever evil Google did.  If it is not working, do a search and replace (look for ^p, and replace with ^p^p; ^p is  ¶ which old timers grew up with in Word).
