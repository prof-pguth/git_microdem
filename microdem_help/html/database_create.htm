Data Base Create

The Record view on the left allows you to edit a single field. The Grid View on the right shows the fields that will be created in the new data base; you can edit these directory on the grid.

Before creating a data base, you must set the number of fields and their properties (name, type, and length). This form allows two views of your proposed data base.

  • Field Name must be 10 or fewer characters, using the letters "A" to "Z", the digits "0" to "9", and the "_".
  • Field type determines how the data is stored, and what kinds of operations you can perform on it.  Examples include String for character data, integer, float, and date.
  • Field length determines how many digits will be stored.
  • Decimals field (number of places after the decimal point) is only required for float fields; it will be ignored for others.

The Insert button takes the current contents on the Record View and copies them onto the Grid view.

Copy DB button allows you to copy the field structure of an existing dBase table.

When you have the correct fields set, select the Create DB button.
