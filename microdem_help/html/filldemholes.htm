Fill DEM Holes

Filling holes or voids makes the DEM prettier, but not more accurate (you cannot recreate data that was not captured in the DEM.

Automatic option when opening a DEM or creating a DTM/DSM from lidar point clouds / creating a DTM; set options on Import/Export  tab of options form.

Manually with "Interpolate across holes (smooth)" on Edit Menu, DEM Holes submenu.

  • Fill holes in the DEM which have missing data. There are very limited cases in which this option will be worthwhile, the primary ones being after import of an ASCII XYZ file or merging of USGS 1:24K DEMs which have holes along the edge (current USGS DEMs should not have this problem), or some Lidar data sets with holes.  The program interpolates in either two or four directions, with an inverse squares weighting. It finds the nearest neighbors only on the same row, same column, or the diagonals as the value to be interpolated

    Set two parameters on Import/Export  tab of options form:

    1. Max Gap to Fill: the program must find neighbors within this distance, or it will leave the hole unfilled. These are in terms of the grid spacing.  The bigger you set this, the farther away from reality the interpolation becomes.
    2. Allow two directional interpolation: if this is set, the program must find two neighbors, in either the NS or the EW or the NE-SW or the NW-SE direction. If this is not set, there must be neighbors in each of the 4 quadrantss, all within the search radius.

This option is not intended to work with very sparse XYZ files. 

Filtering the DEM, and allowing the filter to proceed with missing points, can produce similar results.
