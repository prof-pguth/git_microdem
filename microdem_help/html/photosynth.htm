Suggestions culled from the web, mainly applicable to Photosynth (no longer available), but probably apply to other software as well:

  • Select a point to keep roughly in the center of your frame for every shot. This point can be obvious and visible or one that you visualize. Think of it as the center of gravity of your spin, and try to keep your camera pointing toward it in all shots.
  • Shoot in landscape mode, and at a wide focal length. Shoot at least 20 to 30 photos, making sure to overlap your shots a lot. If you are trying to capture something with many holes, shoot 40 or more photos as you move around the object to reduce 3D artifacts.
  • If you shoot all the way around an object, do not inadvertently spiral in or spiral out when completing the loop. The easiest way to complete a clean loop is to put a marker on the ground where you shot the first frame, then make sure you shoot the last frame from the exact same spot.
  • Turntable Shots: The rule of thumb for turntables is to make anything that doesn't rotate disappear. In practice, this means that any non-rotating background needs to be as featureless as possible.
  • Photosynth wants corner-like things and blob/speckle type things to track. Reflective surfaces and largely single color expanses probably won't synth well.

All of this software has crashed; you might need to get the commercial program (it has a demo version, and a reasonable academic license), or experiment to find out if the problem is the software, you not following directions, or badly configured imagery.

One of the challenges is that the computer science and computer vision communites has a different axis convention (the z axis comes out of the page or screen), while the GIS convention has the z axis vertical, normal to the earth's surface.  You must insure that you understand the difference, and keep your coordinate systems straight.

VisualSFM

Online download at  http://ccwu.me/vsfm/ 

  • You can get the CUDA or non-CUDA version.  CUDA will be faster, but could have issues depending on your graphics card (upgrade the drivers)
  • Need to download and  put cmvs.exe/pmvs2.exe/genOption.exe [pthreadVC2.dll] into the same directory as the main exe file.
  • Steps (http://ccwu.me/vsfm/doc.html#  shows the icons for each step )
    • Add images
    • Match images
    • Sparse reconstruction
    • Dense reconstruction.  You can use SURE for a second opinion on this step.
  • Tutorial
  • http://ccwu.me/vsfm/doc.html#errors

Additional options for manipulation and reconstruction

  • Sure: http://www.ifp.uni-stuttgart.de/publications/software/sure/index.en.html .  Much better quality output than VisualSFM, but slow and huge output files.
    • Open NVM file from VisualSFM
    • Project, start processing
    • Setting the Parameters, Level/Resolution to the left will greatly speed up operations and make the resulting file size much more reasonable.
    • Provides LAS files in the 3D_Points directory
    • Provides a PLY file in the Mesh directory
  • Multi-View Environment http://www.gcc.tu-darmstadt.de/home/proj/mve/  (not working fully)

Agisoft Matashape (formerly Photoscan) is the commerial standard.  I had a student use it.  We got better results with the free software, but that could have been because we learned from the issues with the photos to optimize the geometry, and we used a commercial DLSR instead of a smart phone.

MeshRecon:    http://www-scf.usc.edu/~zkang/software.html , http://zhuoliang.me/meshrecon.html 

  • Copy or rename old "img" and "results" folder first if you want to save the last project
  • Put images in "img" folder
  • Run the "run.bat" command
  • Resulting PLY mesh will be in the "results" folder.
  • No luck so far in getting colors on the mesh
  • Can run out of memory; use fewer pictures, or change the level of detail (see the readme file)
