Fields in the old DEM header record:

  • DEM Type: data sets can be either UTM-based, with points on a regular rectangular grid, or lat/long-based, with points at regular intervals on the geographic grid. Because of earth curvature, the two types of data behave differently over large areas.
  • DEM Precision (Z data type): short float (4 byte), integer (2 byte), or byte.  You cannot change this.
  • Digitizing Datum for DEM: the datum used for creating the data. If you are creating a new DEM from a paper map, the datum should be listed on the legend at the bottom of the map. The program can transform the data from this datum to the datum you need for your map.  If your datum is not specifically listed (and only the NAD and WGS datums are directly listed), you want the local datum which will be the default datum you have selected for MICRODEM .  The "rectangular" datum should be used only as a last resort, and it treats the coordinates only as a Cartesian grid and has no way to get the latitude/longitude.
  • Z units: the units for elevations (or gravity, magnetics, etc) in the data.
  • X-Y units: the units for the grid. Valid responses are dictated by the DEM type.
  • Columns: the number of columns in the data set. Columns run north-south, so the number of columns (along with the data spacing) determines the width of the data set.
  • Rows: the number of rows in the data set. Rows run east-west, so the number of rows (along with the data spacing) determines the height of the data set.
  • X-Spacing: distance between points, which might differ from the y-spacing. 
  • Y-Spacing: distance between points.
  • SW x coord: in either UTM or Lat/Long coordinates, depending on the type of DEM, for the SW corner.
  • SW y coord: in either UTM or Lat/Long coordinates, depending on the type of DEM, for the SW corner..
  • UTM Zone: the 6° zone used for UTM coordinates. Zone 1 centers on 177°W, and each succeeding zone is 6° to the east.
  • Max Z: extreme high value.
  • MinZ: extreme low value.

MICRODEM uses MaxInt (32767) to mark missing data, for both integer and floating point data sets.

Record and type definitions needed to read the header record and DEM in the MD format. All conventions follow Object Pascal, from Borland International. The records are not word aligned but packed. The format changed in Dec 2002. Old files will continue to be read, but new files will break other software using the format. There are 2 changes:

  1. The DefProjName, which links to a PRJ file describing the projection used with the DEM. This was instituted for Finnish DEMs on the KKJ projection. Until it is extended more generally, this will not be further documented, but it offers the potential to deal with DEMs using any map projection.
  2. The addition of floating point fields for the SW corner and the x and y spacings. These were required for the 1/3" NED from the USGS. You can recognize a new DEM because of the 0 in the LatInterval and Long Interval fields, which have now been renamed as iLatInterval and iLongInterval.

Samples of how to check for the new or old header:

  • if (BaseXUTM24K = 0) and (BaseYUTM24K = 0) then begin //new header
    • BaseLat := fSWCornerY;
    • BaseLong := fSWCornerX;
  • end
  • else begin
    • BaseLat := BaseYUTM24K / ArcFactor;
    • BaseLong := BaseXUTM24K / ArcFactor;
  • end;
  • if (Headrecs.iLatInterval = 0) and (Headrecs.iLongInterval = 0) then begin //new header
    • LatOrYInterval := Headrecs.fLatInterval;
    • LongOrXInterval := Headrecs.fLongInterval;
  • end
  • else begin
    • LatOrYInterval := Headrecs.iLatInterval;
    • LongOrXInterval := Headrecs.iLongInterval;
  • end;

  • tSpacingUnit = (SpaceMeters,SpaceSeconds,SpaceMinutes, SpaceKM,Space100m,SpaceFeet, SpaceKFeet,SpaceDegrees, SpaceHundredthSecond,MercProj100m, PolarStereo100m,Space10m,SpaceTenthSecond,SpaceTenThousandthDegree,SpaceIntFeet,SpaceUSFeet);
  • tZValueUnit = (Meters,Feet,TenthMgal,Milligal,TenthGamma,Decimeters,Gammas, HundredthMGal,DeciFeet,Centimeters,OtherElev,HundredthMa, TenthPercentSlope,Undefined,zDegrees,UndefinedHundredth,lnElev,LogElev,,zTenthDegrees,
    UndefinedThousandth,HundredthPercent,UndefinedTenth,HundredthDegree,
    Tens,Hundreds,Thousands,NLCD2001,LandFire,Nanotesla,NLCD1992,IntegerCode);
  • tDigitizeDatum = (WGS72d,WGS84d,NAD27d,NAD83d,Spherical,Local, Rectangular,LamAzEqAreaSphere,SinusEllip,PRDd,UK_OS_grid,ddDefined);
  • tDatumCode = string[5]; //Pascal, 6 bytes since byte[0] is the string length
  • tEllipsCode = string2; //Pascal, 3 bytes since byte[0] is the string length
  • tDMAMapBasicDefinition = packed record //for the DEM header, only the last two fields are (or may be) currently used, but others are loaded from the datum/ellipsoid data files and used inside the program
    • h_Adat,
    • h_f : double; {added July 95}
    • h_XDat,
    • h_YDat,
    • h_ZDat : SmallInt; {added July 95}
    • h_DatumCode : tDatumCode; {added June 98}
    • h_EllipsCode : tEllipsCode; {added June 98}
  • end;

 tHeaderRecord = packed record //for a DEM, version 3, introduced 22 May 2010

  • DEMUsed : tDEMAvailable;
  • DEMPrecision : tDEMprecision;
  • DataSpacing : tSpacingUnit;
  • MaxElev, {max elevation in data set}
  • MinElev, {min elevation in data set}
  • fLatInterval, {y spacing}
  • fLongInterval : shortFloat; {x spacing}
  • OldNumCol,OldNumRow : word; //removed 6/16/2011, so you can get larger grids
  • ElevUnits : tZValueUnit;
  • DefProjName : string[7]; {Pascal, 8 bytes, byte[0] is string length} {complex projection definition}
  • SquareID : string2; {Pascal, 3 bytes, byte[0] is string length}
  • UTMZone : Byte; {6 degree UTM Zone number, USGS/MGRS standard: 1 = W177, 60 = E177}
  • DMAMapDefinition : tDMAMapBasicDefinition;
  • DigitizeDatum : tDigitizeDatum; {sets datum for DEM, and it is transformed to the desired local datum for use}
  • LatHemisph : AnsiChar; {N or S}
  • CreationInformation : string[29];
  • FIPS_Zone : tInteger16;
  • hdfSWCornerx,
  • hdfSWCornerY : Float;
  • NumCol,NumRow : integer; //added 6/16/2011, so you can get even larger grids
  • FutureUse : array[1..7] of byte;

end;
