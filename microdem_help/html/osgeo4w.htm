OSGEO4W

OSGeo4W download:  If at all possible do this from the ethernet, and not Wifi. You will wind up with a 2.6 GB directory, and the installation takes some time. If the installation has problems in the middle, the intallation will probably be corrupt and you will probably have to manually delete the entire c:\osgeo4w folder and download again. Once you have installed the package, MICRODEM will find it and will perform all the GDAL operations.  You might have to restart MICRODEM if it was open during the install so that it will find GDAL.

Detailed directions for download:

  • You do not need admin rights on your computer to install
  • Edge work arounds when during the 2021-2022 academic year Chrome would not work, and Edge was very particular.  These do not seem to be needed any longer, but are kep here because of the serverity of the problems what were encountered. 
    • To "cannot be downloaded securely" pick "See More"
    • On the line for the osgeo42 download, pick the "..." and then Keep
    • Pick "Keep anyway"
    • Pick "Open File"
  • Pick Express Install, which is not the default.  If you pick the custom install and go elsewhere with the location of the installation, you will have to correctly locate GDAL directories which is difficult for some people
  • Pick any of the available servers
  • Pick all four packages
  • Next button to accept the unmet dependencies
  • Accept multiple licence terms, probably around 4
  • Sit back and wait (might be as little as 6 minutes on the ethnernet with little network congestion), or a full period (or even longer) on the wifi if shared with a large class.
  • The "Progress" status bar and other status bars and the percentage in the window caption will restart multiple times, so don't get your hopes up too soon.   
  • At the end, for the first time (8/28/2022) the install asked to make changes to the registry.  It is unclear which program requires this, and if the program could run from an external hard drive and be shared among multiple computers
 

Sofware included, with some options directly from MICRODEM

  • GDAL--can do a lot of things.  MICRODEM now uses it to convert many formats to a Geotiff (floating point) that MICRODEM can read.
  • QGIS--an open source GIS program. 
  • SAGA.  Can be called from QGIS.
  • GRASS.  Can be called from QGIS.
  • libLAS
  • PDAL, which requires a custom installation so it will be included.

To just get GDAL: http://trac.osgeo.org/gdal/wiki/DownloadingGdalBinaries (not recommended, and has not been tested recently).
