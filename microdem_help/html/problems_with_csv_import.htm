Problems with CSV Import

  • The CSV format is ill-defined, and while it works well with basic data, complicated cases can cause problems.
  • Non-Western characters can present problems.  If you do not need the columns with what could become illegible text, delete them in Excel before you import.
  • Commas can present problems, being mistaken for column delimiters.  Delete them with a global search and replace in Excel, or replace them with a ';'.  You can put the commas back once you get the file imported into MICRODEM.
  • Missing data can be symbolized with a letter, character, or "-".  All of these will make the field a string, and limit your options with that field.  To avoid that, do a global search and replace in Excel to make them a blank or a magic number like -9999.
  • There can be extraneous text in the file.  Usually this will be at the top, the bottom, or somewhere on the right.  Try deleting columns to the right, or rows on the bottom.  If you cannot easily find it (Notepad can be easier than the spreadsheet to spot problems on the right), copy the actual data portion to a new worksheet in Excel, and use that.
  • The spreadsheet is likely to remove a leading 0 for ZIP (New England) and FIPS codes ("A" states).  This causes problems when you want to Join databases.
    • Add leading 0's to the field (Edit database, String field operations, add leadings zeros)
    • Remove leading 0's from the field (Edit database, String field operations, Remove leadings zeros).  While this can be the easiest solution, it will leave you with both databases having a non-standard FIPS or ZIP code.

  • String fields can sometimes come in centered, with extra blanks before and after the text. Edit database, String field operations, Trim blanks.
