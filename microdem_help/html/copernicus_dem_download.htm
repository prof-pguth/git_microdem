Copernicus DEM Download

The easiest way to get the Copernicus, though not without some serious caveats, is from OpenTopography (Directions)

  • Metadata: https://portal.opentopography.org/datasetMetadata?otCollectionID=OT.032021.4326.1 This changed in July 2024 after I reported questions.  You should read this very carefully to undertsand what is happening.
  • Tile level metadata will be stripped out.  For most users, this will not be significant, because the metadata is hard to use, and greatly increases this size of the data.
  • As of 7/2025, they are using the DGED 2023 1 version.
  • Raster bulk downloads use AWS command line. 
    • If you want the complete data set and have the disk space and download capacity, this is the best option at present.
    • The overlapping rows and columns will be stripped out so, so the tiles in the one second DEM will be 3600x3600 .   For most users, this will not be "significant" but you should carefully consider if this applies to you (it should have no effect in MICRODEM).
    • The variable longitudinal spacing will be present.
    • There is no way to see which version of CopDEM you are getting.  Most tiles have not changed, so this might not affect you (but could).  It's also not easy to get the changed files from the official download.
    • It is at aws s3 ls s3://raster/COP30/ which is different from the AWS option below
  • Graphical selection browser
    • For the latitudes where the longitudinal spacing changes from 1", the data will be reinterpolated to 1"x1" when you use the cookie cutter option.  This will affect some options, like slope.
    • It is shifted to pixel-is-area instead of pixel-is-point.  There are "significant" elevation differences when you try to do a difference map between official CopDEM and this version; it is unclear if this is just the 1/2 pixel offset.
    • File names are not helpful
    • Limit grid to size you can get in the OpenTopography interface

AWS had the COP data, at https://registry.opendata.aws/copernicus-dem/

  • It has 3600x3600 pixels in the 1" data, trimming the overlap rows and columns
  • It has 3600x3600 pixels at all latitudes, meaning elevations have been interpolated at higher latitudes. 
  • It says it is pixel-is-area
  • It shifts the origin a half pixel in each direction
  • This is some but not all of the changes done by Open Topography (above) and a different AWS location from OpenTopography
  • I have not figured out (or tried very hard) how to actually get any of this data

New official download

  • login with user account.  Until you do, you get no results and no error message
  • https://explore.creodias.eu/
  • Have to pick the sensing or production date, moving one month at a time
  • Under COP-DEM, need to pick desired option e.g. DGE_30.  You do not want the DTED option.
  • Download will be named DEM1_SAR_DGE_30_20110108T161631_20121218T161105_ADS_000000_IWc0.DEM.zip
  • After unzipping, the actual DEM will be several levels down:
    • DEM1_SAR_DGE_30_20110108T161631_20121218T161105_ADS_000000_IWc0\
    • DEM1_SAR_DGE_30_20110108T161631_20121218T161105_ADS_000000_IWc0.DEM\
    • Copernicus_DSM_10_N51_00_E023_00\
    • DEM\
    • Copernicus_DSM_10_N51_00_E023_00_DEM.tif"

Panda (1/29/2024 status)

  • https://panda.copernicus.eu/web/cds-catalogue/panda
  • Sign in to your account
  • Choose the datasets option
  • Pick the COP version (get DGED)
  • Outline  box
  • Search
  • Select all
  • Have not yet  cracked the authorization to download products

Old version, no longer being updated:  Downloads at https://finder.creodias.eu/ 

  • login with user account
  • Pick collection (COP-DEM), Instance (GLO-30), and Product Type (DGED_PUBLIC)
  • Select polygon (lower left)
  • If you upload a KML outline, it will not change the map until you select "Search" (very bottom left)
  • Search products
  • Download directions: https://creodias.eu/-/how-to-order-products-using-finder-api-  (all Greek to me)
  • Download options
    • One at a time, so this is only feasible for small areas
    • Zip names look like: DEM1_SAR_DGE_30_20110423T040654_20121013T041307_ADS_000000_kTTf.DEM.zip which are more or less random from what I see
    • Unzipping gets folder that looks like: Copernicus_DSM_10_S27_00_E019_00, which has the location of the SW corner
    • The DEM itself will be "Copernicus_DSM_10_S27_00_E019_00_DEM.tif", about 50 MB for the 1 sec DEM, in a folder with a total size of about 104 MB.
    • Copy as paths/urls does not produce an ASCII result
    • Add to cart option is not available
