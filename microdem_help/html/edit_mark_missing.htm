Edit, Mark as Missing

Edit menu submenu.  These work on single points, on the DEM/grid currently in memory

  • Selected range: select actual values that will be removed.
  • Outside selected range:
  • Single value:
  • Everything except single value
  • Selected percentile range:  fast way to remove extreme values
  • Outside selected percentile range
  • Everything above cutoff:
  • Everything below cutoff:
  • Isolated value in hole
  • Isolated Value All Neighbors Different (must be a byte grid)
  • Isolated values, specify neighbors same
  • Multiple values: the grid must have integer values, and you can only remove those in the range 0-255. You enter values into a memo box, and can choose to either delete those that match a value in the list, or to retain those that match a value in the list.
  • Byte data, 0 to missing: if you have a byte grid where 0 is missing, this will make the missing values equal to 255, and lower all the other values by 1.
  • Excessive slopes: if the slope is extreme, it is likely to result from an erroneous elevation (say a bird strike in a lidar DEM).  You pick the slope that is too high to be believable, and the buffer of pixels around the point to remove.  Then you must deal with the holes in a second step.
  • Land: removes all values where z > 0.1; intended primarily for the ocean
  • Water: removes all values where z< 0.01; intended primarily for the ocean
  • SRTM water bodies
  • Non-USA regions: the accuracy of this depends on the file you use, and the scale of your map compared to the mask file.
  • Land cover categories
  • Outside lat/long box: enter the NW and SE corners of the region you want, and all points outside the box (defined by lat/long) will be marked as missing.
  • By column/row
  • Matching grid from file: you can remove either valid or missing points from the second grid.
  • Matching open grid; you can remove either valid or missing points from the second grid.
  • Mask all other grids to match this one.  All points in the other DEMs that do not have a valid value in this one will be set the missing.  The DEM will then be resaved with just the area with valid data.  The original DEM will be lost, so if it is needed, operate only on a copy.
  • Selected color--remove all pixels with a particular color, but must be careful if that color is also used on the map.
  • All but selected color
  • Interactive adjusment
  • Flood fill

Other options that perform similar functions:

  • Mask Grid option on  Raster GIS menu can mark points as missing, using a range of values in a reference grid.
  • Mask DEM with shape file provides another way to mark points as missing.
  • Map masking.
  • Filters can remove points that do not have neighbors, but create a new grid in the process.
