Database Symbolization

Databases can be opened:

  1. Solely for cartographic display.
  2. For full GIS use, including
    • Filter--restrict the database based on values of attributes
    • Map query -restrict the database based on geographic locations
    • Identify data base records.  Double click on a feature, and see its attributes from the data base table.

Databases can be symbolized:

  1. Database (shapefile) and then display each database individually
    • Symbol and color selection.  This  has limited ability to use different symbols within a single database, unless you fill COLOR or ICON fields in the database.  You can save the default symbology for display whenever you open the database.
    • Options on Plot button popup menu.  This allows a range of options, but they cannot be saved for later automatic display.
  2. Produce a database with the databases to symbolize, filters to use, and the symbology.  You can create this database on the Database menu and then open atop a base map from the Database menu .
    • Load cartographic overlay: opens database with a list of layers solely for use as a map overlay.
    • Load DB grouping: opens each database, with its table visible.
  3. Load project which will open databases and use the stored Symbol and color selection.  This allows you to load a number of databases

Some databases have default symbolization which you can customize:

  1. Gazetteer
  2. TIGER
  3. OpenStreetMap
  4. Natural Earth world outlines.
