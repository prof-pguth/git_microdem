LAStools Resampling

Data Manipulation, Resample, LAS lidar

LAStools--you must have downloaded and located these programs for them to be used on Point Cloud data.  The programs used here do not have a restrictive license (others do have a restrictive license).  MICRODEM will call LAStools for these conversions:
  • Assign projection.  You must know the correct projection used by the data provider; using this option will overwrite any existing projection information.  You should look in the metadata to get the projection code; a search for the term "EPSG" might be the best approach.  NotePad++ is a good program to view metadata.  You can use the LAZ file as input, and the conversion will be a LAS file.
    • Assign projection by EPSG: you must enter the correct EPSG.  Unless this is UTM or geographic, for MICRODEM you will then have to use "Specified to UTM". 
    • Assign projection UTM zone: you need to know the UTM zone and hemisphere.
    • Assign projection by EPSG and reproject to UTM.
  • Lasinfo: show the metadata; if particular
    • Reproject, Geo to UTM: MICRODEM must correctly identify the file as Geographic, and will call LAStools to convert to UTM in the zone for the file.
  • Reproject.  The file must already have a correct projection; if not you must first use the assign option.
    • Reproject, Specified to UTM: if LAStools can identify the projection, it will convert to UTM.  It can be fooled.
    • Reproject, Specified to geo: if LAStools can identify the projection, it will convert to geographic lat/long.
  • Lasinfo you want to see if the data has a projection.  NotePad++ is a good program to view metadata if the information in the las header is insufficient.
  • XYZI to las: takes an ASCII file, and creates a LAS file.  The extension must be the fields in the order in which they appear (XYZ, XYZI, and so on).  The operation does not expect a header, but will ignore it.

If LAStools cannot identify the projection, you can use the command line version of LAStools to do the conversion.  You must get the projection information from the metadata (perhaps from lasinfo, or File Metadata on the Analyze menu) and enter it.  Then when the projection is defined, you can use the batch conversion in MICRODEM.

If this is working, you should see the black CMD window while it processes.  If it not does not work, you should get the command  from the debug log or the window that reports the problem, and paste it into a CMD prompt window, and see the error message from LAStools.

LAStools can also Compress/Uncompress point clouds.
