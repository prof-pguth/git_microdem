LOS/Weapons Fan Algorithms

Unless you are investigating the effects of the different choices in the LOS algorithm, most of these parameters should probably not be changed.  The exception might be if you must really tweak the algorithm for maximum speed, and are willing to compromise accuracy.

The one parameter you might want to change is the Fan Zoom Factor.  See the discussion under Mixed visible/masked cells.

Invoked from the fan parameters form.

These parameters apply to both the LOS and the weapons fan:
  • Vertical earth curvature algorithm
  • Fan (viewshed) drawing method (applies only to fan and not LOS)
    • Ray spacing
    • DEM spacing multiple and Map spacing multiple (for Radials, full coverage),
    • Point spacing (for Radial lines, discrete)
  • Radial from zoom map size: this should provide an improved version of the Radials, Full coverage.  When we are sure that it works, we will probably remove the older method.  Uncheck this only if it gives you problems.
  • Point selection method (along LOS)
  • Viewer and target on grid node
  • Point interpolation method for intermediate points not on a DEM grid node. You should change this only after extremely careful deliberation. This is not used used in the Yoeli or Breshenham methods.
  • Horizontal earth curvature
    • Switchover distance: (applies only to smart horizontal curvature)
  • Closest blocking distance:  points closer than this distance will not block the line of sight.  This corrects for slight errors in locating the observer, and for problems with the DEM. A reasonable value for this might be 1-2 times the DEM spacing.
  • Missing data blocks LOS.
  • Fan zoom factor: controls the accuracy (and speed of drawing, which are inversely related).   See the discussion under Mixed visible/masked cells.

 

The only options we recommend changing would be, under specified conditions:

  • Fan drawing method
    • Radial lines, discrete.  This can speed up results when you only need a reasonable estimate of the coverage from a location, and do not need a detailed map.
    • Radial, full coverage.  Recommended for most operations.
  • DEM spacing multiple 
    •  Recommended to be 0.5, but can go to 1 without much degradation.
  • Map spacing multiple 
    •  Recommended to be 0.5, but can go to 1 without much degradation.
  • Horizontal earth curvature
    • Over short distances with lat/long DEMs, (out to 10 km for 1/3" or 10 m DEMs), you might get a noticeable speed up by using DEM Grid or Geographic Lat/long coordinates for the algorithm.
