Data Converters/Translators/Viewers

Must Have

  • OSGeo4W: there is now only the 64 bit network installer (MICRODEM is also only 64 bit), and install the desktop version of the package for GDAL in the default location.  You will wind up with a 1.6 GB directory in the root directory of C.  
  • DNRGPS--convert GMX and KML to shape file.  You want "unprojected shape files" for the output. It also deals with getting data from your GPS.
  • 7z.exe and 7z.dll
  • Cloud Compare
  • Google Earth
  • NotePad++ : good program to view metadata and other XLM files.  It has line numbers, tabs for multiple files, and the data will be present when you close MICRODEM.

Grid converter

  • http://www.geospatialdesigns.com/gridconvert.htm: You can export Surfer (not used by MICRODEM) to the following formats which are useful: ArcView ASCII (*.asc), ArcView Float (*.flt with *.hdr), or X,Y,Z triplets (space or comma delimited) (*.dat or *.csv).  You best choice here is probably the GridFloat, but the datum/projection will be an issue.

GeoMapApp: http://www.geomapapp.org/index.htm (can export in Fledermaus format).  GeoMapApp is an earth science exploration and visualization application that is continually being expanded as part of the Marine Geoscience Data System (MGDS) at the Lamont-Doherty Earth Observatory of Columbia University.

DXF files:  Nasty format, not completely GIS-compatible; avoid if at all possible.  We have not tried it recently (think years, if not decades).

  • Converters
    • Use CAD2Shape:  (http://www.guthcad.com.au/cad2shape.htm), but had some issues with the file we tried (but still beat other options we could find in about 2010)
  • Viewers
    • DWG True View 2010 (from AutoDesk at http://usa.autodesk.com/adsk/servlet/pc/index?id=6703438&siteID=123112 )
    • A9CAD (http://www.a9tech.com/) does not view 3D
    • Fledermaus does not do complete data import, but does get the 3D.
    • ArcGIS did not get complete 3D import last time we checked
  • GDAL has capabilities
    • MICRODEM shell
      • The export contours to shapefile on the Calculate menu can create a DXF; make sure you pay attention to the caveats on the input and output.  We have not tested the output to see what programs like it.
      •  Resample GDAL/OGR vectors may be able to read some
    • QGIS or command line invocation of GDAL are also possible
