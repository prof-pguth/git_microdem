MICRODEM Basics

MICRODEM is a standard Windows program and inherits functionality from the operating system.

Help file--the downloadable CHM file has an index and full text search.

 

  • Open map:  File, Open menu choice or a button on the Main Program Toolbar.  You must know and identify the type of data. The most common are:
    • New DEM or other grid. For a DEM, the default display is a combination reflectance map with color elevation tinting.  Right click on the map window, and change the display parameter, or adjust the "reflectance options".  You can change the default DEM map type on Maps tab on options form.
    • New image for satellite imagery or air photos.
    • Quick vector map
    •   Open shapefile map. Vector data sets can be opened on their own, but are most often displayed as Database (shapefile) overlays on another data set.
  • MICRODEM opens maps with what it considers to be a reasonable set of default settings, showing the entire data set on screen without scrollbars. 
  • Selection actions require a double click. 
  • Right clicking on the map  or other window brings up selected options most relevant for the current window settings.
  • Menus: there will generally be a single menu, with options depending on the type of window that is active.
  • Toolbars: there can be multiple toolbars
    • Main Program Toolbar options do not apply to a particular map
    • Map toolbar buttons which apply only to that map. The number of toolbar buttons that appear on the map window toolbar depends on the map size and the screen size.  On a small laptop screen, you might have the use the menus or right click alternatives to the buttons; almost every operation can be started in several ways.
    • Database table controls
    • Other windows, like graphs, can also have their own toolbars.
  • Setting map size
  • Map Overlays
  • Map enhancement.
  • Map zooming, especially Subset (graphical) & zoom.
  • Copy to clipboard
  • Force screen redraw

 

    Saving changes or edits

    DEM:

    • DEMs are loaded into memory for fast use.
    • Changes are not saved unless you explicitly Save the DEM.  If the DEM has never been saved (for example, after you Create a derivative grid, it should be marked as unsaved on the map title bar).
    • If you have made edits, that status will also be indicated.
    • You can undo edits which have not been saved by reloading the DEM on the Edit menu.
     

    Database:

    • Changes to a database take place immediately. 
    • If you want to be safe, create a backup before you begin edits.
    • If the database title bar indicates it is unsaved, the database has been written to c:\mapdata\temp, and will be erased when you close the program.  This is used for what the program considers to be scratch files; if you want to save them, save them to a different location.

 

  • Consider using map library database.
