  • Art Loc: computes coordinates for each artefact.  This will be in a text file.  Save it. Import this file, in CSV format, into a database, which you can then close.   Then open the artefacts data, and Link/join  it to a database with the artefact locations. Then use the option on the edit menu, Map Coordinates, Add XYZ from joined table  to add the XYZ locations.

OpenGL option

Three different slices open at the same time.  You can select a new slice with the right radio group, and pick its orientation with the left one.

Cluster (from SaTScan).  Use the "Clusters" button, and a CLUSTERS.DBF file in the directory with the project files. This needs the fields 'X', 'Y', and 'RADIUS'.

We are using these data sets with X-Y-Z coordinates in meters, with an arbitrary datum.  Many normal GIS functions do not yet work, and may be added over time.  It is probably possible to crash or hang the program by picking an inappropriate action.  We will either fix or disable the option when we hear about it.

These data sets are huge.  We have an octree subdivision working (Edit button on the database table form), but have not finished implementing it (index, selection, merging on the fly).  At the moment that is a very low priority.

You should consider using the thin option when you do displays or analysis for these data sets.

For files converted to DBF format (which will be less efficient than using the Point Cloud Option);

  • Display with 3D graph option on  Stats button on database table form. 
  • Export files as Text (CSV) export on Report button on database table form.
