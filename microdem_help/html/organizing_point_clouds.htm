Downloading and Organizing Point Clouds

Getting point clouds

  • I recommend getting the point clouds, and not the derived grids that many sites (OpenTopography or  https://coast.noaa.gov/dataviewer/?#/ ) offer.  You can create your own grids from the point cloud, and have more options for hillshading. If you just want the grids, don't bother with the point clouds.
Download lidar data
  • Suggested download locations (mostly US) and links to international Country sites.
  • You will likely need LAStools, which MICRODEM can call for reprojection and projection assignment
  • If the data does not have a projection, you must assign it.  Assign projection by EPSG and reproject to UTM, which you can do with a LAS or LAZ file.
  • If the data is not in the UTM projection, you must reproject the data to UTM.
  • If you get LAZ files and you did not need to assign the projection or reproject, you will need LAStools to uncompress LAZ--point clouds 
  • If you are unfortunate enough to get data in the proprietary zLAS format, you need the ESRI tool to convert to LAS

Each point cloud should go in its own directory

  • Insure the directory has an easy to understand name.  The program will refer to the point cloud by the directory name.  You can also open the cloud with the folder.
  • The point cloud is likely to have many files (hundreds are quite possible), so it can be easiest to open by directory rather than files
  • Open by directory will find files nested in subdirectories
  • If you get data from OpenTopography using the graphical selection, every point cloud will have the same file name, and be huge.  You should subset the cloud with Data Manipulation.  As you zoom into the data, you will not need to open the files that are not on the map, which will speed up operations, but operations can be threaded and processed in parallel.
  • You can open up to 5 clouds, and show them separately or together.
