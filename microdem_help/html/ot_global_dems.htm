  • Graphical interface option in browser
    • Pick a region that includes what you want.  It is better to get too much rather than not enough and then having to come back.
    • Open the Global Data Tab and choose the data set and the resolution for those that have 1" and 3" data.  Get the geoid version for the Vertical datum unless you are absolutely sure you want the ellipsoid version.  If there is any chance you might get both, store them differently as the downloads do not indicate the vertical datum.
    • If you are having problems with the graphical selection, you can manually select the coordinates.  You can use the Bounding box that covers the area you want.
    • You do not need a hillshade (uncheck the option) since the GIS can easily create. 
    • The limit is 450,000 square km. 
    • Keep the default Geotiff format.
    • Download will be a tar.gz file, which is a Unix format you can decompress in MICRODEM.  You can have the program delete the compressed file.  The file will be in a sub-directory in the folder where the tar.gz file was.
    • If you let the download change the file name, say by adding " (1)" because there is another download with the same name, you will have to manually untar the result from the un-gzip (so don't let Windows rename).
    • File will be named "output_srtm", so you must keep track of which version you got. The metadata has the data spacing, but not the vertical datum.  Considering renaming the final TIF file with the geographic region and the Vertical datu selected.
  • Raster bulk downloads use AWS command line
    • This gets the complete data set, so be ready for some serious bandwidth and hard disk requirements
    • I suspect you could figure out how to get just the area you want, which is left as an exercis for the curious student.

You must have GDAL installed, via OSGEO4W, to use the DEMs in MICRODEM.

The grids use -9999 for missing data without indicating this in the Geotiff header.  You should mark it missing on Import/Export, or editing the grid, if the color scheme is distorted by the invalid large negative elevations. 

DEM results will be in tar.gz (Unix) compressed file.  Compress/Decompress option in MICRODEM will use decompress with 7-Zip without bothering you with the details.  If you do multiple downloads you must segregate them in different directories, as they decompress with the same names and some of the decompressions will fail.
