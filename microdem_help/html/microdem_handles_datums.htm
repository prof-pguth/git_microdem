How MICRODEM Handles Datums

A DEM's header record indicates the datum upon which it was digitized. These choices are geared toward US data, using either NAD or WGS, and will provide the greatest flexibility for datum shifts and display of dual datums.   The options for digitizing datum:

  • WGS72
  • WGS84
  • NAD27
  • NAD83
  • Spherical
  • Local--in this case you must be sure to select the correct datum for the map display, as the program will not perform a datum transformation, but will corrrectly translate between UTM coordinates and lat/long.
  • Rectangular--no chance to convert the coordinates to lat/long

When the program displays coordinates, it will display using the primary (and optionally the secondary) datum you have selected, which do not have to be the digitizing datum.  Thus the datum on which the DEM was created can be completely hidden to users.

If you export as

  • ASCII DEM, it will use your primary display datum--this means you might be getting a datum transformation and not the original grid locations. The ASCII file does not indicate its datum.
  • DTED, the datum will be transformed to WGS84.  This is both explicitly listed in the file and implicit in the DTED specs.
