MICRODEM Significant Version changes

Program History

Minor changes, bug fixes, and modifications at the source code level are not referenced.

The completeness of this log varies, and currently is very sparse.  The important improvements will now be listed in the Program History, organized by functional aspects of the program, and linked to the relevant section in the help file, and the MD forum hosted by Delphi will list bug reports and other questions that have been asked.

The current build numbering system uses a number like 2014.6.24.1, where each number has a meaning:

  • Year,
  • Month,
  • Day,
  • Build during that day.

The old numbering system used a number like 12.34.28.4, where each number had a meaning:

  • First number, the major version and corresponds to the version of Object Pascal compiler used. 
  • Second number, the minor version.  This increased every time a complete install was posted on the WWW or every month.
  • Third number, the release.  This will correspond to the day of the month the version was compiled, and can be used to track debug installs.
  • Fourth number, the build. 

Version numbering is a Windows standard, stored in the executable file and retrieved with an API call.
