Gridding Lidar

Lidar surveys create point clouds, essentially random collections of 3D points.  While data analysis directly with the point cloud is increasing, conversion to regular grids generally greatly decresases the data storage requirements, and allows the use of standard raster GIS tools.

 

Prior to gridding, the user must select the desired final surface or surfaces, generally DTM or DSM.

High and low noise can be flagged, so it is not used, or removed from the data set.

The point cloud can be classified, to at least identify the ground returns, and only these will be used for a DTM.

For a DSM, all points can be used, or the electrical power lines, and vehicles could be removed (neither of these are routinely included in the point clouds distributed by national mapping agencies).

There are two general methods for creating the grids:

  • Triangulation

    • Points are triangulated with a Delaunay triangulation.
    • At each of the desired grid nodes, find the triangle containing that point, and then interpolate to find the grid elevation.
    • If there are many points per grid cell, many of the triangles will not be needed.
    • Works best with a small number of point, suggesting small tiles.
    • Edge effects will be minimized with larger tiles, or with a buffer of points around the tile, which are only  used for the triangulation of point within the tile.  This will slightly increase the data storage requirements.
    • There will be no gaps within the region outlined by data points.
    • If there are grid cells with no points from the lidar survey, they will appear in the final grid as obvious triangular facets.  The larger these areas, the more obvious the effect.  These could be eliminated, either by not interpolating in large triangles, or by editing the grid to remove any points with no close neighbors, which can be done as a fast raster operation.
Circled anomalies from the triangulation in a DSM from Denmark.  These result from the lack of water reflections in the castle moat.

  • Drop in the bucket

    • Very simple and fast
    • Each point is assigned to the nearest point in the desired grid. 
      • For a DSM, if the point is higher than the current point at the grid cell, the new point replaces the older point.
      • For a DTM, if the point is lower than the current point at the grid cell, the new point replaces the older point.
    • The selected point could be mislocated in the grid by half the grid spacing.
    • The gridding can interpolate across very small voids, and the fills will not be noticeable.
    • For a DTM or NVS, the more points in the grid cell, the greated the chance that at least one is actually on the ground.  This will reduce the number of tree trunk spikes.
Two Non vegetation surface grids.  The 1 m grid has a number of anomalies, because the point density of the lidar survey did not reliably deliver a point on the ground in every grid cell, and has a number of spikes.
