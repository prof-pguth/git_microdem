MICRODEM's Design Philosophy

  • The program should run entirely from the GUI.  It is hoped that moderately sophisticated, but not GIS-fluent users, can use the program, and that it can be used in education where the goal is visualizing and interpreting data which is more important than developing GIS-literacy.
  • A number of options can now be called from the command line
    • Parameters
    • Batch mode Conversions
  • All options should have a reasonable default value, and then the user can modify is desired.
  • We should not reinvent the wheel, and should use free tools where appropriate, where it is easy to link to them from Delphi.  Exceptions to this include:
    • We want to support the Android and MacOS (and potentially iOS) versions for common operations, so we will try to avoid libraries that do not support all these platforms.
    • When we need raw speed.  Using the tools can require writing the DEM to a standard format Geotiff, and then reading the new Geotiff returned by the tool.
    • When we have good working, legacy code written before the free tools became available.
