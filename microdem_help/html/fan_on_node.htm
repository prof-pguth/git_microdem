Viewer and Target on Grid Node

Forcing the observer or target locations to lie on grid nodes in the DEM, while not explicitly stated, appears to be used in the computations for some algorithms. Algorithms that pre-compute intervisibility for an entire DEM use only the grid points, and blowing up some programs shows a pixelated appearance. With high resolution DEMs this would not be a problem, but with typical military DEMs having 1" or 3" resolution, this might not allow for optimal level of detail.

Fisher (1993) discussed this issue, and found that it had the largest impact on the final analysis.  He noted that the viewshed could be considered Point-to-Point, Point-to-Cell, Cell-to-Cell, or Cell-to-Point. In the case of Cell-to-Cell, Fisher advocated going from each corner of the viewer’s cell to each corner of the target cell.  Potentially 16 separate calculations would be required, although the algorithm could stop with the first visible finding.  Of the seven GIS implementations in Fisher (1993), five used a grid cell location for the viewpoint, one used a vector point, and one allowed either representation.
