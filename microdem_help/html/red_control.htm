Redistricting Control Form

 

Redistricting Form.

Statistical data for the current redistricting plan. The size of the color bars are proportional to the target population shown in the "Goal" column, which should be 100% (or very close) for each district.

For each district, it shows the number of blocks, the population, and how close each district is to the goal. 

Change redistricting plan:

  • Combo box  shows the district that will be selected with the graphical operations.
  • Force redraw, which can be slow.  If you use Quick pick, you can select multiple blocks to move and then do a quick redraw for all.  The status bar will show how many blocks have not been redrawn.
    • Pick one: move individual blocks into the currently active district by double clicking on the map. The district will be recolored and statistics updated.  Use this for fine control.
    • Pick box: drag around a rectangular region with mouse, and move all blocks inside the box into the currently active district by double clicking on the map. The district will be recolored and the statistics updated.  Use this for rapid, large changes.
  • Evenness controls the imbalances that will be highlighted in color on this form.  Districts shown in green exceed the goal, and those in red do not have enough population.    Those in black are within the evenness tolerance factor.  Your goal is to have all districts in black.
  • Redraw: 
    • Districts on the map. They will not be automatically redrawn when the map is resized or modified.
    • Percentage black
    • Percentage Hispanic
    • Population
    • Population density
    • Percent black
    • Percent Hispanic
  • Report: creates HTML file with the results from the redistricting table.
  • Districts:
    • Create new districts: give each a name and color.
    • Clear all districts--removes district assignments and colors
    • Assign to districts
      • Randomize--these will be scattered and not contiguous, and present a good start for complete gerrymandering.
      • Vertical bands
      • Horizontal bands
    • Create new district
    • Recolor current district
  • Help:
  • Highlight--show high density block for ethnic/racial categories. 
  • 1 Race Checking "1 race" calculates WHITE_PC and BLACK_PC using the total in the single race categories WHITE1, BLACK1, and ASIAN1.  Unchecking it uses the total population POP, which includes smaller racial categories and the multiracial people.  HISP_PC always uses the total population because the Hispanic category is in addition to race, and every respondent must answer Yes or No to the classification.
  • ID button on map toolbar can show population of individual blocks.
  • Plot>  Color code by DB field lets you view the population statistics in any way that you want.  Scaling (quantile)  might make the maps easier to visualize.


Highlight:  Checking radio button "Black" or "Hispanic" changes district to show those blocks that have greater than a 50% (or other selected cutoff selected to the right) minority population.  If one of the goals is to create a district with a majority that is over 50% Black or Hispanic, then obviously you need to target the blocks in solid as opposed to the others.  If none of the blocks in district have a majority of the minority group, then neither can the district as a whole, but you can look at smaller percentages.
