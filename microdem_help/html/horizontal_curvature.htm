Horizontal Earth Curvature

Horizontal earth curvature determines how to compute the “straight line” path between the observer and the target. For simplicity, this is probably generally computed in the DEM grid coordinates, as shown in figures discussing the algorithms (e.g. Yoeli, 1985; Champion and others, 1995). For intervisibility over short distances this should make no difference, but over long differences it will compute a different sight line for a UTM or a geographic DEM.  This effect does not appear to have been prominently discussed in the literature. 

These results suggest at least four ways that could be used to compute the straight line required for intervisibility: UTM coordinates, geographic coordinates, DEM grid coordinates (UTM or geographic depending on the type of DEM used), or accurate geodetic computation (Vincenty, 1975).  From the desired end points, the intermediate points can use any desired coordinate system.  This will be easiest with radial interpolation, but could probably be done with the other methods.  Experiments show that UTM and geodetic coordinates produce almost identical results, but the geographic coordinates differ substantially.  UTM coordinates can be used for short sight-lines, where the conformal projection preserves the azimuths, but for longer distances the distortions will be become noticeable.  UTM coordinates can probably be used until the deviations for the correct geodetic line exceed the DEM grid spacing.  Geographic coordinates can only be used for very short sight-lines—meaning that the currently available DEMs for military use, DTED and SRTM, cannot use the coordinate scheme depicted in diagrams.  Furthermore, the point interpolation schemes that use the sides of grid squares and triangles will not be easy to code because the desired path curves in the geographic coordinate space and finding the intersections along the sides will not be simple.

 

Regardless of the type of DEM, each algorithm can be used to calculate the straight line and the coordinates of evenly spaced points along the profile can be computed in any desired coordinates—UTM, geographic, or the DEM grid which will be related to one of the first two.

 

The suggested "Smart" algorithm uses UTM coordinates out to a crossover distance, at which point it switches to geodetic computations.  UTM coordinates will be faster, but for very long distances or at the edges of UTM zones may lead to distortion.

 

Over short distances with lat/long DEMs, (out to 10 km for 1/3" or 10 m DEMs), you might get a noticeable speed up by using DEM Grid or Geographic Lat/long coordinates for the algorithm.
