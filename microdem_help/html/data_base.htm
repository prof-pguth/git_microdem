Data Base Plotting & Querying

Open with

  • Button on Map window toolbar. 
  • Database, Load menu option

When opening the database, the program can show or hide different file formats.  The default settings will show both the DBF and the SHP files for ESRI Shapefiles, because MICRODEM allows self-contained DBF files for point databases.  If you will not be using those, you can elect just to see full shapefiles by showing only the SHP file.  If both files are shown, it does not matter which you pick; the program will correctly open either.

Formats allowed

  • ESRI Shapefile and usage in MICRODEM
    • Download an incredible number of shapefiles from the WWW.  If they are not in Lat/Long coordinates, you will have to convert them. This should now be automatic if you have GDAL installed.
  • DBF dbase Table, (VOLCANOES.DBF is the test file included with a full installation of MICRODEM)  with
    • fields LAT and LONG, floating point
    • any number of other fields
    • the sample data base has 1500 volcanoes, with names, elevations, and type
    • create your own data base, or import a file to create a database.
  • CSV file (points only), which meets the rules for a dBase table. If you open a CSV file, it will be converted to a dBase table and in the future you should open that file or you will lose any changes to the database after you have opened it. Problems with CSV import
  • TXT (points only)
  • KML  (points only, for others Import with DNR GPS )
  • GPX
  • Garmin FLT files

The program presents a table with the data and options.
