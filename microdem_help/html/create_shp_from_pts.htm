Create area/line shapefile from points

Option on  "Report" button of GIS table display window.

You must first convert the data into a dBase table or point shape file, which must have the following fields (all others will be ignored):

  • LAT (decimal degrees, west negative)
  • LONG (decimal degrees, south negative)
  • Whatever other fields you need to filter the data.
  • A field to name the records if you want to split the file based on combined data.

Excel is probably the easiest way to do this, with an export to CSV, and then open with the DB load option in MICRODEM.

This will create area or line shape files, with one record per file.  It will include all currently active point records in the database file, so you should filter the database appropriately before you use this option.

 

You can:

  •  3D shapefile: create a 3D shapefile if there are elevations in the database.
  • Multiple records based on filter: filter the database by the values of a field, and create a separate line record with each filter.
  • Thin: take only every nth point in the file
  •  Start/end times: put the start and end time of each line into the DBF table.
  • Line end points; put the lat/long of the start and end of each line into the DBF table

You can add fields to the database later, or Merge multiple shapefiles.
