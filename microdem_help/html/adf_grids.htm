GDAL

GDAL is a translator library for raster and vector geospatial data formats that is released under an X/MIT style Open Source License by the Open Source Geospatial Foundation.

Get GDAL installation from OSGeo4W.  If you want to try a direct download, you have to configure it on your own.

For the most common formats, GDAL is called directly from:

  • Open Grid/DEM
  • Open image

Less mainstream formats are converted before use:

  • Data Manipulation,  Resample:
    • GDAL rasters:
    • GDAL/OGR vectors:

A number of complex formats cannot be directly read by MICRODEM:

  • ADF ESRI grid format.  Pick the "w001001.adf" file (which will be the only large file in the directory, and not the small "w001001x.adf" file).  An ADF file, like a shapefile, actually contains a number of files which always have the same names (the directory name is what is unique), and all files in the directory must be kept together.  Failure to do so will cause problems. 
  • JPEG2000 (JP2) compressed file; pick the JP2 file. 
  • ECW wavelet compression format for imagery.
  • IMG grid format from ESRI for imagery and DEMs.
  • GeoPDF
  • USGS SDTS
  • NetCDF (for complex files, import might be problematic)
  • Tiled or compressed Geotiff.

Other options implemented with GDAL

  • Subset TIFF imagery, keeping multiband
  • Merge imagery

MICRODEM creates a command line or batch file  to use GDAL.  This command will be written to the debug log.  If there are problems, you can copy that command into a CMD window, run it, and see the error messages to try to debug the process.

Reference: GDAL/OGR contributors (2020). GDAL/OGR Geospatial Data Abstraction software Library. Open Source Geospatial Foundation. URL https://gdal.org
