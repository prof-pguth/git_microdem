Managing Lidar Data

Do not use spaces anywhere in the path to the lidar data.  Do not ignore warnings about this.

Place every survey in its own folder, with a short descriptive name, which the program will use to identify it, such as "DUCK_2015" and "Duck_2018"

If you have multiple surveys of a single area, organize them under a single higher level folder such as "Duck_surveys"

Many surveys produce data only in very long strips.  These are more cost effective comparted to regular "mow the lawn surveys", but lead to very long and skinny maps, with mostly missing data.  These are difficult to visualize

  • Utility line corredors
  • Beach erosion surveys, where all the change is along the coastline.

When visualizing or displaying multiple surveys, think about what you want to show, and control this with the checkboxes:

  • All surveys on the same map or 3D view, or in the same derived grid.  This makes sense if you have adjacent surveys with each cover only part of what you want, or you need the second survey to fill in holes in the first, and do not care about temporal differences.
  • All surveys separately.  This makes sense if you are tracking change over time.

If you get a warning that your data does not have projection information, or has an unsupported projection:

  • Look at the metadata from the LAS file, to see what the projection is if it is present.  SPCS means it will have to be reprojected
    • See if there is a metadata file in your download folder.  It could be XML, TXT, or HTML
    • Analyze, File Metadata. LAS--you can find if the problem is with MICRODEM, or the data files.
      • via MICRODEM--this will be faster, as it just reads the header
      • via  Lasinfo--reads the entire file, and provides some statistics
  • Look at the web site from the data download  for the file or project metadata
  • LAStools file resampling.  Based on what is actually in the data, you must pick the correct option:
    • Unsupported projection, Reproject, Specified to UTM:
    • No projection information, Assign projection.  If it is not UTM (where you need the UTM zone) or geographic, you need to get the EPSG code.
      This will create new LAS files with "_utm" in the name.  If you did not recycle the original files, you need to remove them for the directory manually if you want to use directory selection.

Geotiff codes, from http://geotiff.maptools.org/spec/geotiff6.html or https://svn.osgeo.org/metacrs/geotiff/trunk/geotiff/html/usgs_geotiff.html.  You need to do a search of 26958, to find it is Florida east.

Image 3

EPSG code from https://spatialreference.org/ref/epsg/?search=florida&srtext=Search  These are the results for Florida, and we already know it was the east zone and in feet, so we want EPSG 2236.  Get the UTM zone for Florida, 17.   We can now use LAStools with Assign projection by EPSG (2236) and reproject to UTM zone 17 N, with the elevations in feet.
