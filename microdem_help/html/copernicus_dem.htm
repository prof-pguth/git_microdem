Copernicus DEM

Copernicus DEM:

  • nominal 90 m, 30 m, 10 m (actually arc second spacing)
  • 30 m and 90 m freely available.  The intention of the f&o license negotiated with Airbus (see here pages 15-17) was to come as close as possible to the general Copernicus licensing policy, which in a few words means: after registration as a user and acceptance (Art. 1) you can essentially do with the data what you want (Art. 4) as long as you ackowledge the source (Art. 6). 
  • Avoid the DTED version (integer is much worse than floating point elevations)
  • Validation report:    https://spacedata.copernicus.eu/documents/20126/0/GEO1988-CopernicusDEM-RP-001_ValidationReport_V1.0.pdf/edc88601-897e-82df-5604-b6b60cbecb57?t=1586527010843
  • AIRBUS (2020) Copernicus DEM: Copernicus Digital Elevation Model Product Handbook  RFP/RFI-No.: AO/1-9422/18/I-LG, Version: 2.1.  Available online at https://spacedata.copernicus.eu/documents/20126/0/GEO1988-CopernicusDEM-SPE-002_ProductHandbook_I1.00.pdf/082dd479-f908-bf42-51bf-4c0053129f7c?t=1586526993604
  • https://spacedata.copernicus.eu/fr/dataset-details?articleId=394198
  • Radar has some penetration in both snow/ice and vegetation

Copernicus DEM metadata quality layers (data format and type), in the AUXFILES directory

  • Codes are in the Land Cover Database
    • EDM: Editing Mask: 8 Bit unsigned integer, GeoTIFF  Page 18 of the Product Handbook has the codes
    • FLM: Filling Mask: 8 Bit unsigned integer, GeoTIFF  Page 19 of the Product Handbook has the codes
  • Other files
    • HEM: Height Error Mask: 32 Bit floating point, GeoTIFF
    • WBM: Water Body Mask: 8 Bit unsigned integer, GeoTIFF
    • ACM: Accuracy Layer: KML vector file
  • Source: https://spacedata.copernicus.eu/collections/copernicus-digital-elevation-model

EU LAEA DEM

  • Precursor to Copernicus
  • https://ec.europa.eu/eurostat/web/gisco/geodata/reference-data/elevation/eu-dem/eu-dem-laea
  • Multi-GB zip files, which uncompress to a multi-GB tiff, which QGIS did not much like (so subset immediately)
  • The LAEA projection is not very standard
  • Probably only useful if you absolutely need a DEM in this projection (to map other EU data)

The names for the DEMs have a lot of variability. 

 

“CPP10_DGED” and “CPP10_DTED” are the Package names for version 2019 of the 30m product

“CPP30_DGED 20” and “CPP30_DTED 20” are the Package names for version 2020 of the 30m product

 

The 2020 version substitutes the 2019 version. Use 2020 always where available. Discrepancies between both versions are not due to time update, but to improvements in the product.

There is also now a 2021- version

Determining the version is not easy.

 

The dates of image acquisition for the DEM are in the Copernicus_DSM_10_N28_00_W018_00_SRC.kml file in the Preview directory. For the La Palma island (Canaries, Spain) this was over about 2.5 years fron 2011 to 2013.
