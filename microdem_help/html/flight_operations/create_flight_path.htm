Create Flight Path File
Use Excel to create a file as shown above. You will probably be importing a TXT file. You must have the fields LAT, LONG, ALTITUDE, and SPEED filled in.  The others can be filled in later.

The order of the fields does not matter.

All other fields (AZIMUTH, Z, DISTANCE, and CLEAR_R, CLEAR_OFFR, CLEAR_L, CLEAR_OFFL, ) need a dummy value which will determine the number of digits allowed in the database.  Use the values above.  The RED and BLUE fields do have values (it does not matter where they are) out of sight on this screen capture.

This is the minimum number of fields for the option to work; other fields can be present and used in standard database operations, but will be ignored for this.

Save the file in CSV format.  You can use dBase if you are feeling lucky, but there are too many chances for problems to risk it.
