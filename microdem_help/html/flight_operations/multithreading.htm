Poor Man's Multithreading

  • Use task manager to insure that a single version of the program does not tax you CPU, RAM, or disk IO.  If any one of them is close to 100%, multithreading is not worth it.
  • Copy the program EXE with a different name.  It should still be in c:\microdem
  • Open the programs.  They will each have their own INI file with the MICRODEM defaults.  If you want to load a specialized set, save a customized one and  then restore it to the others.

This works on any operation in the program, and does not require rewriting to run in parallel
