Navigation Menu

Submenu on "EDIT" button of database table form.  The button is only enabled if you check the box on the right of the row of buttons, since all changes are permanent (save a copy of the database if you are concerned about editing mistakes).

  • Add travel path (Azimuth, distance, speed).  To get speed, there must be a DEC_DAYS or DEC_HOURS field.  The records must be in chronological order, in a point database.  You can select to add the speed, distance, cumulative distance (for instance useful if you want a graph of a parameter versus distance traveled), and to record in meters per second or km per hour. There will be no entries for the first record, except for a 0 in the cumulative distance field.
3D distances will compute the distance on the hypotenuse for each segment, as well as the gains and drops in elevation.
  • New DBF with speed/path by ID:  Must have fields LAT, LONG, ID, SHIP, MONTH, and DEC_YEARS.  Will create a new database, with the readings halfway between each pair of readings, and adding fields HEADING, SPEED_KPH, AND KNOTS.  Designed for the CLIWOC data base.
  • Get azimuth from magnetic heading: convert magnetic heading to true north, using the WMM model.
  • Add Distance/Azimuth fields
    • Database must have fields "LAT", "LONG", "LAT2", and "LONG2".
    • If the data base does not already have "DIST_KM" and "AZIMUTH" fields, they will be added.
    • Program will compute the distance (in km) and direction from each point in the data base to the other point with the "LAT2", and "LONG2" coordinates.
  • Get azimuth from magnetic heading:  requires fields LAT, LONG, ALTITUDE, MAG_HEAD  and will create and compute AZIMUTH  (the magnetic heading plus the declination).
  • Dead reckoning
