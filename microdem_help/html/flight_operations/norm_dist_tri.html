Distance Normalization For TRI

The original definition of TRI made no account for the different horizontal distances to the 8 neighbors, in particular longer distances for the diagonals, and the differences in EW and NS spacing for arc second DEMs. As such TRI was very similar to Average Adjacent Neighbor slope algorithm, but the similarity was disguised by using only the diffeence in elevation and not dividing by the horizontal distane.

MICRODEM offers several ways to normalize so that all 8 computations use the same horizontal distance.

  • None--this will match other computations for TRI, with minor differences for how they handle missing and edge data.
  • Use the elevation difference as a directional slope derivative, and use a linear estimate of the elevation at the specific distance from the central point.  For the diagonal difference with a geographic grid, this will not necessarily be at a 45 degree angle (DEM information table will show the actual angle).

  • Interpolate, bilinear: find the point on the diagonal that is  1/2 * sqrt(2) pixels out on the diagonal.  For UTM grids, this provides essentially the same results as the directional derivative.

TPI and MAD2k use the same normalization.
