New in Version 3

Release 3.0 of TerrbaBase II and build 11.01 of MICRODEM appear summer of 1998. Major changes include:

  • Program became less DEM-centric. MICRODEM reflects the program’s DEM heritage. While it has done image analysis for some time, use of the image registration and ground coordinates have required that a DEM for the area be available.
  • Same map window used for both DEMs and images. 
  • "Image" refers to both satellite images and scanned maps. Satellite images should be viewable in both map and image windows, but the scanned maps should only be visible in a map window.
  • Data sets are used in native format, and do not have to be imported into the program’s unique formats. This has been true to some degree for DEMs, but is now true almost universally for both DEMs and imagery. This greatly reduces the chances for registration problems.
  • The program uses datum and ellipsoid files from TEC's GEOTRANS.
  • New types of raster data sets supported: ADRG, CADRG, some GEOTIFFs, and at least one flavor of NITF.
  • Display DXF and SDTS DLG vector files.
  • USGS SDTS DEM data handling  improved.
  • GPS options  improved and expanded, including route replay
  • The terrain fly-by has been greatly enhanced.
  • Many map and image display options significantly speeded up.
  • Quality of perspective view  greatly improved.
  • Print to high resolution printers (HP Lasers at 600 dpi, large plotters like the HP650C) has been improved.
