Derivative Grids

Invokes popup menu, which can be selected in two places:
  • Calculate Menu, Derivative Grid
  • Raster GIS menu, Create derivative grid

This will create a new grid, with the same size and spacing as the original grid.

You must save the new grids (File, Save DEM)  if you want to use them again.  When you save, you can choose either the MICRODEM or GeoTIFF format.  The derivative grid can be manipulated in its own right.'

The program can automatically save the files if your pick the Multiple parameters option, which will be savbed to a subdirectory "geomorph_grids" in the directory with the original DEM.  If you want to convert to GeoTiff, either:

  • Open and then File, Save DEM, and pick Geotiff.
  • Data Manipulation, Resample, DEM format, convert to Geotiff, which allows conversion of all DEMs in a directory.

Some options (relief, openness) require that a region size be selected. The size is in meters, and converted into the number of grid points needed.  The computations will use a square centered on the point.

Some options (e.g. slope, reflectance, aspect) can be selected from the display option of the DEM display, but have limited modification capability since they are just display modes of the DEM.

  • Multiple parameters--This allows creation of multiple grids while you do something else.  Parameters within each grouping are computed together, and doing more than adds little time, but will increase memory usage.
  • Slope--uses current slope algorithm
    • All slope/aspect
    • Slope (%)  (100 * tan)
    • Slope (%)
    • Slope components (x and y) (directional derivatives)
    • Degrees
    • m/m
    • Sin
    • log10 tangent
    • ln tangent
    • sqrt sin
    • slope and roughness
    • Pixel areas
  • Aspect--uses current slope algorithm
    • Aspect 360
    • N-S component: sin(aspect), 1 is east, -1 is west
    • E-W component: cos(aspect), 1 is north, -1 is south
    • Aspect difference: largest difference in aspect in the NS, NE-SW, EW, or NW-SE directions, with a value 0-180. Choose 3x3, 5x5, and 7x7 windows.   Can help to pick out valleys and ridges
  • Curvature:  Use with caution.  There are also options with external tools.
  • Relief
    • Relief, average elevation, elevation standard deviation, REL (percentage of area that is lower), and TPI: all for a specified region size
    • Relief, Summit levels, Base Level, Geophysical Relief
    • Immediate neighborhood drop: drop off is the distance from the point to the lowest point in the box with the selected size.
  • Topographic ruggedness/roughness
  • Convergence index
  • Topographic ruggedness (RRI, TPI, TRI)
  • Openness
  • Reflectance
  • Terrain category: pick a terrain category, and the points meeting the criteria will have a single value in the resulting grid.
  • Transform elevations
    • Meter elevations
    • Ln (natural log).  Cannot have negative values.
    • Log (base 10) transform elevations Cannot have negative values.
    • Normalize parameter
    • Parameter percentiles
  • Terrain classification--uses current classification algorithm
    • Ridges--all peaks and ridges will have the elevation; other points will be missing
    • Valleys--all streams will have the elevation; other points will be missing
    • All terrain categories--will have an integer code for the category
  • Iwashashi and Pike Classification
  • Terrain organization maps: Organization (s2/s3), Flatness (s1s2, inverse slope), Fabric direction, Roughness factor (slope)
  • Moments for region (elevation, slope, two curvatures, the moments for a selected region size)
  • Point density: divides value at each grid point by size of the grid; the result will in count/m².
  • Optimal viewshed location
