Terrain Analysis Tutorial in MICRODEM

Insure MICRODEM has been installed.  If necessary, download here.  You should immediately download the help file from the help menu.  MICRODEM is a Windows 64 bit program.

If the version of the program is older than the revision date at the bottom of this page, update the program.

Introduction to key operations in the program

Go to File, Introductory tutorials, Harper's Ferry Terrain Analysis.  This will download the following data sets, a 100+ MB file, unzip it, and open the DEM.

Data Set Scale Uses Format Web Source
DEM: NED, 1/9" 1/9" DTM, about 2.66x3.43 m Terrain characterization Geotiff USGS, National Map Viewer
OpenStreetMap: vector overlays   Cultural awareness Shapefile http://download.bbbike.org/osm/
Lidar Point Clouds 2.8 pts/m² 3D visualization LAS USGS, http://earthexplorer.usgs.gov/
Scanned geologic map   Interpret geology KMZ USGS OF-2000-297, from http://ngmdb.usgs.gov/ngmdb/ngm_catalog.ora.html

This data uses common open formats, and should work with no problems in commercial ESRI ArcGIS or open source QGIS.  These data sets will show the range of operations available in MICRODEM; you can later deal with the challenges of getting additional data.  This region was chosen because its relief highlights the 3D possibilities in GIS.

This tutorial emphasizes the terrain analysis aspects of MICRODEM; the program can do many other things.

Investigate displays possible with the DEM.  Display parameter lets you pick the main option, and then many have additional choices that you set the first time you pick or you can modify later.

  • Colored hillshade or shaded relief is the default.  This will generally provide better appreciation of the terrain than an Elevation map
  • Grayscale hillshade provides a good base map, on which you can overlay additional data sets in color.
  • Slope shows the terrain steepness; the gray scale reversed option provides a nice base map.
  • Openness provides a similar display, and might work better for urban or open terrain lidar DSMs.  This option provides a quick default display.
  • Aspect show the downhill direction, and is important for soil development, vegetation, or ski slope siting.
  • Contours can show the traditional relief depiction.  You can also overlay contours on any base map with Map Overlays

Overlay raster data on the map

  • Overlay, Imagery (KML, KMZ, world files), and pick the KMZ which will overlay a scanned geological map.  You can adjust its opacity.

Overlay vector data on map

  • Database menu, Load OSM overlay, and pick the directory with the OSM data.  This data set varies in quality with the local crowd sourcing community, but generally will have pretty good roads, and sometimes railroads, water, buildings, and natural landcover.
  • Map Overlays allows you to change the opacity of the layer.

3D map operations

  • OpenGL 3D view.  This will be best with a larger map, perhaps even larger than will fit on screen.
  • Perspective views
  • Panorama view
  • Line of sight and Topographic Profiles:  these are similar and show a 2D slice through the terrain.  Line of sight shows intervisibility, important for many military operations, and communications.
  • Weapons fan/viewshed

Open lidar point cloud

  • LIDAR Point clouds and pick the directory with the lidar data.  This is a relatively low resolution survey, but still creates a 300 MB file.  It also has not beeen completely classified (there are no buildings or trees identified), which is typical for many readily available point clouds.
    • Look at the display options.
    • Slice: EW and NS slices through the point cloud.  These are xz and yz.  The xy view will probably be less informative.
      • OpenGL 3D option. 
    • Create grids, Ceiling + floor grids to make a DSM (ceiling) and a DTM (floor). A 1 or 2 m grid spacing is appropriate for this data.  The 1 m grid will be 4 times larger.  The DTM includes the buildings, because removing them requires a building classification to be done on the point cloud, and then replacing the building with a value determined from the surrounding ground.  You can see this in the NED DSM, although it has a smaller scale.   MICRODEM does not attempt to perform these operations, in part because we think the DTM with buildings is actually more useful in most cases.
      • Create a HAG or CHM grid by doing the difference between the DSM and DTM. Use Raster GIS, Multiple grid arithmetic, Difference between two grids.  You want DSM-DTM so the values will be positive.
